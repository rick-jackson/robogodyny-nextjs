# Налаштування Supabase для РобоГодини

Цей документ містить покрокову інструкцію з підключення Supabase до проєкту РобоГодини.

## 1. Створення проєкту в Supabase

1. Перейдіть на [supabase.com](https://supabase.com) і увійдіть у свій акаунт (або зареєструйтесь).
2. Натисніть **New Project**.
3. Виберіть організацію, назвіть проєкт (наприклад, `robogodyny`).
4. Виберіть регіон (найближчий до вас, наприклад, Europe West).
5. Встановіть сильний пароль для бази даних.
6. Натисніть **Create new project** і зачекайте ~2 хвилини.

## 2. Отримання ключів API

Після створення проєкту:
1. Перейдіть у **Settings** → **API**.
2. Скопіюйте:
   - **Project URL** (наприклад, `https://xxxxx.supabase.co`)
   - **anon public** ключ (для клієнта)
   - **service_role** ключ (для серверних операцій, **не публікуйте його!**)

## 3. Створення таблиць у базі даних

Перейдіть у **SQL Editor** і виконайте наступний скрипт:

```sql
-- Таблиця профілів користувачів (ролі)
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  full_name TEXT,
  role TEXT CHECK (role IN ('employee', 'admin', 'superadmin')) DEFAULT 'employee',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Таблиця робочих записів
CREATE TABLE work_entries (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) NOT NULL,
  date DATE NOT NULL,
  start_time TIME NOT NULL,
  end_time TIME NOT NULL,
  work_type TEXT CHECK (work_type IN ('onsite', 'business_trip')) NOT NULL,
  location TEXT,
  is_driver BOOLEAN DEFAULT FALSE,
  has_travel BOOLEAN DEFAULT FALSE,
  travel_time_minutes INTEGER DEFAULT 0,
  comment TEXT,
  rate_used NUMERIC DEFAULT 180,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- RLS (Row Level Security) для безпеки
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE work_entries ENABLE ROW LEVEL SECURITY;

-- Політики для profiles
CREATE POLICY "Users can view own profile" ON profiles
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Admins can view all profiles" ON profiles
  FOR SELECT USING (
    EXISTS (
      SELECT 1 FROM profiles 
      WHERE id = auth.uid() AND role IN ('admin', 'superadmin')
    )
  );

-- Політики для work_entries
CREATE POLICY "Users can view own entries" ON work_entries
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own entries" ON work_entries
  FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own entries" ON work_entries
  FOR UPDATE USING (auth.uid() = user_id);

CREATE POLICY "Admins can manage all entries" ON work_entries
  FOR ALL USING (
    EXISTS (
      SELECT 1 FROM profiles 
      WHERE id = auth.uid() AND role IN ('admin', 'superadmin')
    )
  );

-- Функція для автоматичного створення профілю при реєстрації
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, full_name, role)
  VALUES (NEW.id, NEW.raw_user_meta_data->>'full_name', 'employee');
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

## 4. Налаштування змінних середовища

Створіть файл `.env.local` у корені проєкту:

```env
NEXT_PUBLIC_SUPABASE_URL=https://ваш-проєкт.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш-anon-ключ

# Для серверних операцій (створення користувачів адміном)
SUPABASE_SERVICE_ROLE_KEY=ваш-service_role-ключ
```

**Важливо:** Ніколи не комітьте `.env.local` у Git!

## 5. Створення тестових користувачів

1. У Supabase Dashboard перейдіть у **Authentication** → **Users**.
2. Натисніть **Add user** → **Create new user**.
3. Створіть користувачів:
   - `employee@example.com` / пароль (роль employee)
   - `admin@example.com` / пароль (роль admin)
   - `superadmin@example.com` / пароль (роль superadmin)

Або виконайте SQL для оновлення ролей:

```sql
UPDATE profiles 
SET role = 'admin' 
WHERE id = (SELECT id FROM auth.users WHERE email = 'admin@example.com');

UPDATE profiles 
SET role = 'superadmin' 
WHERE id = (SELECT id FROM auth.users WHERE email = 'superadmin@example.com');
```

## 6. Запуск проєкту

```bash
npm install
npm run dev
```

Відкрийте http://localhost:3000/login

## 7. Функціонал після підключення

- **Авторизація**: Сторінка `/login` з email/password.
- **Ролі**: 
  - `employee` — тільки календар
  - `admin` / `superadmin` — календар + адмін-панель
- **Дані**: Всі записи зберігаються в Supabase (замість localStorage).
- **Адмін**: У адмін-панелі можна переглядати записи всіх співробітників.
- **Додавання користувачів**: Адмін/суперадмін може створювати нових користувачів через Supabase Dashboard.

## 8. Додаткові рекомендації

- Для production використовуйте Row Level Security (RLS) політики (вже налаштовано).
- Додайте email confirmation у Supabase Auth settings.
- Для superadmin можна додати можливість змінювати ролі інших користувачів.

Якщо виникнуть питання — звертайтесь! 

Проєкт тепер повністю працює з реальним бекендом Supabase.
